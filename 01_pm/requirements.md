## プロジェクト基本方針

- アプリ名: StrideEdge（競馬予想支援アプリ）
- 対象: 個人利用前提
- 有料APIは使用しない
- 法令・利用規約・robots.txt・公開条件を遵守する
- 非公式API・規約違反スクレイピング・ログイン突破は禁止
- 本アプリは「予想支援」であり、利益保証・断定表現はしない
- 自動投票・自動購入は実装しない
- ローカル環境で完結することを最優先とする

## Phase 4（実データ連携）スコープ

- スクレイプログ管理（scrape_logs テーブル）

- スケジューラー自動実行（Phase 4は手動トリガーのみ）

## 実装最優先方針（修正版仕様より）

1. **スクレイパーのHTTP取得とパースを完全分離**し、テスト可能な状態にすること
2. **取得した生データの型変換・バリデーションはPydanticモデル層で完結**させること（サービス層・DB層に未加工文字列を持ち込まない）
3. **DB保存は1レース単位でトランザクション管理**し、一部保存失敗時にゴミデータが残らないようにすること
4. **既存ダミーデータ運用を壊さず**、段階的に実データへ差し替え可能な構造にすること
5. **netkeibaへのリクエスト間隔には、必ず1〜3秒のランダムなsleep処理を挟むこと**（通信制限対策・必須）

## ⚠️ netkeiba アクセス絶対条件（Gemini評価反映・設計上の制約）

以下は交渉不可の絶対条件。実装・レビュー・テストのすべてにおいて遵守せよ。

| 条件 | 詳細 | 違反時の対応 |
|------|------|------------|
| **リクエスト間隔** | 1〜3秒のランダム sleep（`random.uniform(1.0, 3.0)`）。固定間隔・0秒は禁止 | 実装拒否・レビュー否決 |
| **1セッション上限** | 最大150リクエスト。超過時は `BlockDetectedError` を raise して処理を停止 | 実装拒否・レビュー否決 |
| **ブロック検知** | HTTP 403 または予期しないリダイレクト（301/302/303/307/308）を検知した場合は即時処理停止 | 実装拒否・レビュー否決 |
| **User-Agent 偽装禁止** | ブラウザ模倣 UA は使用可。ただし偽のクローラー名を名乗ることは禁止 | 実装拒否 |
| **過度なアクセス禁止** | 1日あたりの同期実行は手動トリガーのみ（スケジューラー自動実行は Phase 5 以降）| 設計審査 |
| **利用規約確認** | netkeiba.com の最新利用規約・robots.txt を確認し、PMが本ドキュメントに記録すること（未完了） | **PMアクション** |

### 判定基準（Pass or Play）

| ラベル | 条件 |
|--------|------|
| PLAY | ev_ratio ≥ 65% かつ ev_spread ≥ 5pt かつ warning_ratio ≤ 50% |
| WATCH | ev_ratio ≥ 52% または（ev_ratio ≥ 47% かつ ev_spread ≥ 3pt） |
| CAUTION | ev_ratio ≥ 42% |
| PASS | 上記未満 |
